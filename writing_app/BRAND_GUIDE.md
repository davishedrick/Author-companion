# Scriptor Brand Guide

## Product Personality

Scriptor should feel serious but approachable, structured but creative, and
professional without sounding corporate. It supports writers who care about
manuscript progress, revision clarity, and trustworthy statistics.

Scriptor should feel:

- Calm.
- Capable.
- Precise.
- Honest about data.
- Encouraging through clarity, not cheerleading.

Scriptor should not feel:

- Cute.
- Gamified.
- Overly motivational.
- Corporate or buzzword-heavy.
- Like productivity theater.

## Voice

Use short, direct sentences. Prefer nouns and verbs that name the work:

- manuscript
- session
- project
- current words
- starting word count
- issue
- revision
- history
- progress
- baseline
- catch-up

Avoid vague energy words:

- awesome
- crush it
- optimize your flow
- productivity journey
- unlock your potential

## Copy Rules

### Buttons

Use verb-first commands.

Good:

- Start writing session
- Start editing
- Create project
- Open history
- Resolve
- Clear binding
- Sync document changes

Bad:

- Let's get productive
- Begin your journey
- Make magic
- Supercharge

### Modals

Modals should state the decision and consequence.

Good:

- Clear binding?
- This document is missing. History stays saved.
- End session?
- Publish project?

Bad:

- Are you sure?
- Oops, something happened.
- Important update!

### Warnings

Warnings must identify what is blocked and what the user can do next.

Good:

- Projects unavailable. Authentication required.
- Could not identify this tab. Refresh Google Docs.
- This session belongs to another Google Docs tab.

Bad:

- Error.
- Something went wrong.
- Try again maybe.

### Empty States

Empty states should explain what is absent and offer a concrete next step.

Good:

- No recent activity yet.
- Create a project to track your progress.
- Resolved issues will collect here once you start closing them out.

Bad:

- Nothing here!
- You have no stuff.
- Start being productive.

### Success Messages

Success messages should be factual.

Good:

- Document connected.
- Verified manuscript size: 10,305 words.
- Session saved.
- CSV exported.

Bad:

- Great job!
- Amazing progress!
- You did it!

## Writing Rules

- Use plain language.
- Keep helper copy one sentence when possible.
- Avoid exclamation marks except rare celebratory completion states.
- Do not imply judgment about negative word changes.
- Do not describe deletion as failure.
- Use "Net: -500 words" rather than "lost 500 words" when neutral framing is
  better.
- Distinguish project progress from words written inside Scriptor.

## Data Language

Scriptor treats word counts as source-of-truth data.

Use:

- Starting word count.
- Current words.
- Since tracking began.
- Net words.
- Verified manuscript size.

Avoid:

- Total written, when the value includes pre-existing manuscript words.
- Words written, when the value is actually current manuscript size.
- Progress, without specifying target progress or session progress.
